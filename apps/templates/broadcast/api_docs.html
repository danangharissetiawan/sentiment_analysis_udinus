Dokumentasi API

Dokumentasi API Broadcast adalah panduan bagi pengembang aplikasi untuk mengintegrasikan sistem mereka dengan aplikasi Broadcast. Ini mencakup informasi tentang bagaimana mengakses data dan fitur aplikasi menggunakan protokol API RESTful. Dokumentasi ini juga mencakup informasi tentang format data yang diterima dan dikembalikan oleh API, serta contoh implementasi dan tes untuk memastikan integrasi berhasil.

Hal ini sangat penting untuk memastikan bahwa aplikasi bekerja dengan baik dan memenuhi standar keamanan dan performa.

Endpoint /api/broadcast/{email}/

Method: GET

Endpoint ini digunakan untuk mengambil data broadcast berdasarkan email yang diberikan. Jika data broadcast ditemukan, akan mengembalikan data broadcast. Jika tidak ditemukan, akan mengembalikan pesan "The broadcast does not exist" dan status kode 404.

Response
200: Informasi data ditemukan. 
{
    "name": "admin",
    "email": "admin@admin.com",
    "message": "udinus adalah kampus terbaik",
    "classification": "Positive"
}
404: Informasi data tidak ditemukan. 
{
    "message": "The broadcast does not exist"
}

Endpoint /api/broadcast/create/

Method: POST

Endpoint ini digunakan untuk membuat data broadcast baru. Parameter email harus berisi string "create". Data yang dibutuhkan untuk membuat broadcast adalah nama, email, dan pesan. Setelah broadcast berhasil dibuat, akan mengembalikan pesan "Data created successfully." dan data broadcast yang baru dibuat, termasuk nama, email, pesan, dan hasil klasifikasi pesan. Jika broadcast sudah ada, akan mengembalikan pesan "Broadcast already exists." dan status kode 400.

Parameter
Nama Tipe Keterangan
name string Nama yang di gunakan. (wajib)
email string Email yang digunakan pertama kali saat melakukan POST data. (wajib)
message string Pesan yang ingin dikirimkan. (wajib)
Response
200: Informasi data ditemukan. 
{
    "success": "Data created successfully.",
    "data": {
        "name": "new name",
        "email": "newemail@mail.com",
        "message": "Saya bangga menjadi mahasiswa udinus",
        "classification": "Positive"
    }
}
404: Informasi data tidak ditemukan. 
{
    "message": "Broadcast already exists."
}

Endpoint /api/broadcast/{email}/

Method: POST

Endpoint ini digunakan untuk memperbarui data broadcast yang sudah ada. Parameter email digunakan untuk menentukan broadcast mana yang akan diperbarui. Jika broadcast tidak ditemukan, akan mengembalikan pesan "Broadcast does not exist." dan status kode 400. Data yang dibutuhkan untuk memperbarui broadcast hanya pesan, sementara nama dan email tetap sama seperti sebelumnya. Setelah broadcast berhasil diperbarui, akan mengembalikan data broadcast yang baru, termasuk nama, email, pesan, dan hasil klasifikasi pesan.

Parameter
Nama Tipe Keterangan
message string Pesan yang ingin dikirimkan. (wajib)
Response
200: Informasi data ditemukan. 
{
    "name": "admin",
    "email": "admin@admin.com",
    "message": "saya bangga menjadi mahasiswa udinus",
    "classification": "Positive"
}
404: Informasi data tidak ditemukan. 
{
    "status": "error",
    "message": "Broadcast does not exist.",
    "note": "Please create a broadcast first.",
    "url": "api/broadcast/create/",
    "method": "POST",
    "data": {
        "name": "string",
        "email": "string",
        "message": "string"
    }
}
